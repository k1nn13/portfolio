Philip KinshuckAR adventure book

Technical Artist | Animator | 3D Modelling

Overview:

Based on the classic poem by Edward Lear ‘The Owl and the Pussycat’, this group project makes use of AR technology blending the virtual and physical world to create a complete immersive experience.

Using a system derived from ‘choose your own adventure books’ the user takes full control of the story by moving physical place holders on the page deciding the final outcome.

Setup:

Implementing Trello to organize and assign tasks combined with version control practice using github enabled our group to effectively work remotely.

Creating the concept by visually brainstorming using an interface design tool Figma made it possible to develop ideas through flow charts creating a solid concept and understanding between group members.

Development:

My first task was to implement animations from Mixamo. By combining Unity's Nav Mesh Agent it was possible to precisely apply motion and position to the character. To apply nav mesh data into a prefab it was necessary to use an experimental high level navigation mesh building component, pre baking the navigation data into the scene. Styled particle system FX where used to blend the virtual and real world by masking the boundary of the scene.



Modelling:

Using Maya I created all the assets for the project. The characters where then uploaded to mixamo and then imported into unity. Applying the rigged models into my custom animation controller allowed me to apply different states of animation linked to logic created by other group members.



Texturing:

After UV unwrapping models in MAYA the fbx files where then exported to Adobe Substance Painer where high quality textures can be applied. Using tri-planar projects, normal and specular texture maps allowed for extra detailing and texture. Experimenting with Substance Designer and following a tutorial I developed a custom procedural feather texture seen in the Owl model.



Reflections:

Overall the project had a positive response and the concept was well recieved however there are still some outstanding issues. This could be seen in the class demonstration when the placeholders occasionally failed to move the virtual objects into the correct position, making the experience not flow as expected. This problem will need to be addressed before demonstrating the project to any industry professional.

The most challenging part of my process was learning to implement animations and apply the nav mesh agent in Unity to the prefab. Overcoming this challange took a long time to research and find a solution however the end result allows the project to be easily developed further and add new levels with a more complex layout.

Finally I would like to thank my group for their effort and support throughout development. We worked together and grew as a team, utilising our skills efficiently and implementing our ideas.
